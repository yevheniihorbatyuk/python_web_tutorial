# Урок 3: Основи Django

## Вступ: Від даних до веб-додатку

Ми зібрали дані, але вони просто лежать у базі даних. **Django** — це високорівневий веб-фреймворк на Python, який дозволяє нам створити веб-додаток для відображення та керування цими даними.

### Архітектура MVT

Django використовує патерн **Model-View-Template (MVT)**:
- **Model (Модель)**: Визначає структуру ваших даних (схему бази даних). Це те, що ми створюємо в `models.py`.
- **View (Представлення)**: Бізнес-логіка. Воно отримує веб-запити, взаємодіє з Моделлю для отримання даних і вирішує, що показати користувачеві.
- **Template (Шаблон)**: Рівень представлення. Це HTML-файл зі спеціальним синтаксисом, який відображає дані, підготовлені Представленням.

## Частина 1: Структура проєкту та додатку

Проєкт Django — це сукупність конфігурацій та додатків. **Додаток** — це самодостатній модуль, який щось робить (наприклад, блог, система керування користувачами).

### Створення структури

1.  **Встановіть Django**:
    ```bash
    pip install Django
    ```
2.  **Створіть проєкт**: Ця команда створює основний каталог проєкту та файли конфігурації.
    ```bash
    django-admin startproject myproject .
    ```
3.  **Створіть додаток**: Ця команда створює каталог для логіки вашого додатку.
    ```bash
    python manage.py startapp quotes
    ```

## Частина 2: Налаштування та конфігурація

Файл `myproject/settings.py` є серцем конфігурації вашого проєкту.

### Ключові налаштування

- **`INSTALLED_APPS`**: Список усіх активних додатків у проєкті. Ви повинні додати сюди свій новий додаток.
  ```python
  INSTALLED_APPS = [
      ...,
      'quotes', # Додайте свій додаток
  ]
  ```
- **`DATABASES`**: Конфігурація для підключення до бази даних (наприклад, SQLite, PostgreSQL).
- **`STATIC_URL`**: Базовий URL для обслуговування статичних файлів (CSS, JavaScript, зображення).

## Частина 3: Моделі та міграції

На цьому етапі ви визначаєте структуру ваших даних.

1.  **Визначте моделі**: У `quotes/models.py` ви створюєте класи, які успадковуються від `django.db.models.Model`. Кожен клас представляє таблицю в базі даних, а кожен атрибут — колонку.

2.  **Створіть міграції**: Міграції — це як система контролю версій для вашої бази даних. Після зміни моделей ви запускаєте:
    ```bash
    python manage.py makemigrations
    ```
    Ця команда створює файл, який описує зміни у вашій схемі бази даних.

3.  **Застосуйте міграції**: Щоб застосувати ці зміни до бази даних, ви запускаєте:
    ```bash
    python manage.py migrate
    ```
    Ця команда виконує файли міграцій та оновлює ваші таблиці в базі даних.

## Частина 4: Адмін-панель Django

Django постачається з вбудованим адміністративним інтерфейсом, який є потужним інструментом для керування вашими даними.

1.  **Створіть суперкористувача**: Щоб отримати доступ до адмін-панелі, вам потрібен обліковий запис адміністратора.
    ```bash
    python manage.py createsuperuser
    ```
2.  **Зареєструйте моделі**: Щоб ваші моделі з'явилися в адмін-панелі, ви повинні зареєструвати їх у `quotes/admin.py`.
    ```python
    from django.contrib import admin
    from .models import Author, Quote

    admin.site.register(Author)
    admin.site.register(Quote)
    ```
3.  **Запустіть сервер і увійдіть**:
    ```bash
    python manage.py runserver
    ```
    Перейдіть до `http://127.0.0.1:8000/admin/` та увійдіть. Тепер ви можете додавати, редагувати та видаляти дані безпосередньо.
