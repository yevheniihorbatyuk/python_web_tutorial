# Документація 03_django_app

Цей документ надає детальний опис `03_django_app`, Django-додатку для керування користувачами, містами та країнами.

## Структура проекту

Проект є стандартним Django-додатком з єдиним додатком `users`.

```
03_django_app/
├── config/             # Конфігурація проекту Django
│   ├── settings.py     # Налаштування проекту
│   └── urls.py         # Основна конфігурація URL
├── users/              # Додаток 'users'
│   ├── models.py       # Моделі бази даних (Country, City, User)
│   ├── views.py        # Представлення для операцій CRUD
│   ├── urls.py         # URL-адреси, специфічні для додатку
│   ├── forms.py        # Форми для створення та оновлення даних
│   └── templates/      # HTML-шаблони
└── manage.py           # Утиліта командного рядка Django
```

## Функціонал

Додаток надає наступний функціонал:

*   **Операції CRUD:** Створення, читання, оновлення та видалення для користувачів, міст та країн.
*   **Відносини:**
    *   `User` належить до `City`.
    *   `City` належить до `Country`.
*   **Дружній інтерфейс:** Простий, заснований на шаблонах, інтерфейс для взаємодії з даними.
*   **Пошук та фільтрація:** Можливість пошуку та фільтрації списків користувачів, міст та країн.

## Моделі

### `Country`

*   `name`: Назва країни.
*   `code`: Дволітерний код країни (наприклад, "US").
*   `population`: Населення країни.

### `City`

*   `name`: Назва міста.
*   `country`: Зовнішній ключ до моделі `Country`.
*   `population`: Населення міста.
*   `founded_year`: Рік заснування міста.
*   `is_capital`: Логічне значення, що вказує, чи є місто столицею.

### `User`

*   `first_name`: Ім'я користувача.
*   `last_name`: Прізвище користувача.
*   `email`: Електронна адреса користувача.
*   `phone`: Номер телефону користувача.
*   `city`: Зовнішній ключ до моделі `City`.
*   `bio`: Коротка біографія користувача.
*   `is_active`: Логічне значення, що вказує, чи є користувач активним.

## Представлення

Додаток використовує класові представлення для всіх операцій CRUD.

*   **Списки:** `CountryListView`, `CityListView`, `UserListView`
*   **Деталі:** `CountryDetailView`, `CityDetailView`, `UserDetailView`
*   **Створення:** `CountryCreateView`, `CityCreateView`, `UserCreateView`
*   **Оновлення:** `CountryUpdateView`, `CityUpdateView`, `UserUpdateView`
*   **Видалення:** `CountryDeleteView`, `CityDeleteView`, `UserDeleteView`

## URL-адреси

URL-адреси додатку структуровані наступним чином:

*   `/users/`: Головна сторінка, яка відображає список усіх користувачів.
*   `/users/countries/`: Список усіх країн.
*   `/users/cities/`: Список усіх міст.

Кожна модель має власний набір URL-адрес для операцій CRUD (наприклад, `/users/create/`, `/users/1/edit/`, `/users/1/delete/`).

## Покроковий посібник

### 1. Налаштування

1.  **Клонуйте репозиторій.**
2.  **Перейдіть до каталогу `python_web/module_10/03_django_app`.**
3.  **Встановіть залежності:**
    ```bash
    pip install -r requirements.txt
    ```
4.  **Створіть файл `.env`** на основі файлу `.env.example` та заповніть необхідну інформацію.
5.  **Виконайте міграції бази даних:**
    ```bash
    python manage.py migrate
    ```
6.  **Створіть суперкористувача:**
    ```bash
    python manage.py createsuperuser
    ```
7.  **Запустіть сервер розробки:**
    ```bash
    python manage.py runserver
    ```

### 2. Використання додатку

1.  **Перейдіть за адресою `http://127.0.0.1:8000/users/`** у вашому веб-браузері.
2.  **Ви побачите список користувачів.**
3.  **Ви можете створювати, редагувати та видаляти користувачів, міста та країни**, натискаючи на відповідні кнопки.
4.  **Ви також можете переглядати деталі** кожного користувача, міста та країни.
5.  **Інтерфейс адміністратора** доступний за адресою `http://127.0.0.1:8000/admin/`.
